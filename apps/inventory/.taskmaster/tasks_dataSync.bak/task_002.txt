# Task ID: 2
# Title: Implement Monday.com API Client
# Status: done
# Dependencies: None
# Priority: high
# Description: Develop a client for interacting with Monday.com's GraphQL API, handling authentication, rate limiting, and pagination.
# Details:
Create a Monday.com API client with the following features:
1. Authentication using API key
2. GraphQL query builder for common operations
3. Rate limiting with exponential backoff
4. Request batching for efficient API usage
5. Pagination handling for large datasets
6. Error handling and retry logic

Implement the following key methods:
- `validateApiKey(apiKey)` - Verifies API key is valid
- `getWorkspaces(apiKey)` - Lists available workspaces
- `getBoards(workspaceId)` - Lists boards in a workspace
- `getBoardColumns(boardId)` - Gets column definitions for a board
- `getItems(boardId, options)` - Gets items from a board with pagination
- `createItem(boardId, itemData)` - Creates a new item
- `updateItem(itemId, itemData)` - Updates an existing item
- `createSubitem(parentItemId, itemData)` - Creates a subitem under a parent

Use modern fetch API with appropriate error handling.

# Test Strategy:
Create mock responses for Monday.com API. Write unit tests for each API method. Test rate limiting behavior with rapid requests. Verify pagination works with large datasets. Test error handling with various error scenarios.
